## 二、规则语法总览

### 2.1 JSOUP Default（默认规则引擎）

#### 基本格式
```
类型.名称.位置@提取类型##正则##替换
```
- **类型**：`class`、`id`、`tag`、`text`（文本内容）、`children`（所有子标签，无需名称位置）
- **名称**：类名、ID、标签名、文本片段
- **位置**：正数从 0 开始（0 第一个），负数从 -1 开始（-1 倒数第一个）；可用 `!` 排除（如 `0!1` 排除第二个）；可用区间 `[start:end:step]`（start 可省，end 为 -1 可省，step 支持负数实现反向）。
- **提取类型**：`text`、`ownText`、`textNodes`、`html`、`all`、`href`、`src` 等。
- **特殊**：列表前加 `-` 可使列表倒序（如 `-ul li`）。

#### 示例
```css
class.odd.0@tag.a.0@text                     // 取第一个奇数类的第一个a标签文本
class.item[0:5]@text                         // 取前5个
tag.div.-1@html                              // 最后一个div的HTML
id.content@ownText##广告##                   // 只取自身文本并删除“广告”
```

---

### 2.2 CSS 选择器（`@css:`）

- 标准 Jsoup CSS 选择器，语法参考 [Jsoup Cookbook](https://jsoup.org/cookbook/extracting-data/selector-syntax)。
- 以 `@css:` 开头，也可省略（直接写选择器）。

#### 常用示例
```css
@css:.book-item h2@text
#content@html
a[href^=http]@href
```

---

### 2.3 XPath（`@xpath:` 或 `//`）

- 以 `@xpath:` 或 `//` 开头，遵循 XPath 1.0 规范。
- 支持轴（`ancestor`、`descendant`、`following-sibling` 等）和函数。

#### 示例
```xpath
//div[@class='content']/p/text()
//a/@href
//ul/li[position()=last()]/a@href
```

---

### 2.4 JSONPath（`@json:` 或 `$.`）

- 以 `@json:` 或 `$.` 开头，用于解析 JSON 数据。
- 支持过滤器 `[?(@.price < 10)]`。

#### 示例
```js
$.data.books[0].title
$.data.books[*].name
$..author
```

---

### 2.5 正则表达式

#### AllInOne（用于列表）
- 语法：`:正则表达式`，只能用在搜索列表、发现列表、目录列表、详情页预处理。
- 例：`:href="(/read.*?html)"` 提取匹配的 URL。

#### OnlyOne（用于单值）
- 语法：`##正则表达式#替换内容##`，只替换第一个匹配项。
- 例：`##第(\\d+)章##$1`。

#### 净化（循环替换）
- 语法：`##正则表达式#替换内容`（省略末尾 `##`），用于正文等，循环匹配替换所有。
- 例：`##广告词|推广##`（删除所有广告词）。

#### 组合使用
```js
"content": ".content@text##\s+## ##免费小说.*##"
```

---

### 2.6 JavaScript 规则

- 使用 `<js></js>` 或 `@js:` 嵌入 JavaScript。
- `@js:` 只能放在规则最后；`<js></js>` 可任意位置，甚至作为选择器的分隔符（如 `tag.li<js></js>/a`）。
- 结果存放在 `result` 变量中。
- 所有 Java 类调用需通过 `Packages.java.*` 或已导入的类。

#### 示例
```js
"name": "@js:result.split(' ')[0]"
"chapterList": "<js>java.ajax(baseUrl + '/list')</js>"
```

---

### 2.7 自定义连接符号（`&&`、`||`、`%%`）

- 仅用于同类型规则之间（不包括 JS 和正则）。
- `&&` —— 合并所有取到的值（并集）。
- `||` —— 以第一个取到值的为准（短路）。
- `%%` —— 依次取数（交叉合并），如列表1取第1个，列表2取第1个，列表1取第2个……

#### 示例
```js
"name": "h1@text||.title@text"          // 优先h1，没有则用.title
"chapterList": "ul.list1 li&&ul.list2 li"  // 合并两个列表
```

---

### 2.8 特殊规则：`{()}` 与 `()`、`@put`/`@get`

#### `{()}` 和 `()`
- 在 URL 或规则中，`{()}` 内的内容会作为独立规则执行。
- 在搜索/发现 URL 的 `()` 中只能使用 JS。
- 在其他地方（如详情页规则）的 `()` 中可使用任意规则（需加标志头，如 `@@`、`@xpath:`、`@json:`、`@css:`，默认 JS）。

#### `@put` / `@get`
- 用于非 JS 规则中存取变量（JS 中用 `java.put`/`java.get`）。
- `@put` 中 JSONPath 不需要引号，其他规则需要加引号。

#### 示例
```js
// 在URL中计算page
"searchUrl": "https://example.com/search?page={{(page-1)*20}}"
// 在规则中存储中间值
"name": "@put:{bookId: $.id}@get:{bookId}"
```

## 三、书源配置详解

### 3.1 基本设置
- **bookSourceUrl**：唯一标识，必须填写。
- **bookSourceName**：显示名称。
- **bookSourceGroup**：分组（可选）。
- **loginUrl**：登录页面地址（需配合 `loginUi` 和 `loginCheckJs`）。
- **bookUrlPattern**：正则，用于自动匹配书源（如 `https?://www.example.com/book/.*`）。
- **header**：全局请求头（JSON 字符串），可包含 `User-Agent`、`Referer`、`Cookie` 等。
- **enabledCookieJar**：是否自动保存 Cookie。
- **concurrentRate**：并发请求限制（如 "1" 表示同一时间最多1个请求）。

---

### 3.2 搜索
- **searchUrl**：搜索地址，`{{key}}` 替换为关键词，`{{page}}` 替换为页码（默认从1开始）。
  - 支持相对路径（相对于 `bookSourceUrl`）。
  - 支持 POST：`"searchUrl": "https://example.com/search,{\"method\":\"POST\",\"body\":\"key={{key}}\"}"`。
- **ruleSearch**：
  - `bookList`：列表选择器。
  - `name`、`author`、`kind`、`wordCount`、`lastChapter`、`intro`、`coverUrl`、`bookUrl`。
  - `checkKeyWord`：校验关键词（如搜索结果不包含则视为失败）。

---

### 3.3 发现（含样式）
- **exploreUrl**：多行或 `&&` 分隔，格式为 `名称::URL` 或 JSON 对象（支持样式）。
  - 简单格式：`男生::/shuku/0_1_0_0_0_{{page}}_0_0\n女生::/shuku/0_2_0_0_0_{{page}}_0_0`
  - 带样式（JSON 数组）：
    ```json
    [
      {"title":"今日限免","url":"...","style":{"layout_flexGrow":1}},
      {"title":"频道金榜","url":"...","style":{"layout_flexGrow":0}}
    ]
    ```
- **ruleExplore**：同 `ruleSearch` 字段。

---

### 3.4 详情页（含预处理）
- **ruleBookInfo**：字段 `name`、`author`、`kind`、`wordCount`、`lastChapter`、`intro`、`coverUrl`、`tocUrl`。
- **bookInfoInit**（预处理）：
  - 只能用正则 AllInOne（以 `:` 开头）或 JS。
  - JS 需返回一个 JSON 对象，对象的 key 对应字段名（如 `{ "name": "圣墟", "author": "辰东" }`），然后在规则中直接写 key 名。
- **canReName**：规则不为空时，允许使用详情页获取的书名/作者覆盖搜索页的。

---

### 3.5 目录
- **chapterList**：章节列表选择器，前面加 `-` 倒序。
- **chapterName**、**chapterUrl**：名称和链接规则。
- **isVip**：VIP 标识（当结果为 `null`、`false`、`0`、`""` 时为非 VIP）。
- **nextTocUrl**：下一页目录地址，支持单个 URL 或数组；JS 中返回 `[]` 或 `null` 或 `""` 时停止。
- **volumeList**、**volumeName**：卷列表及名称（可选）。

---

### 3.6 正文（含 WebViewJs、资源正则）
- **content**：正文内容选择器。
- **nextContentUrl**：下一页正文地址（类似目录下一页）。
- **webJs**：在 WebView 中执行的 JS，必须有返回值（非空），返回值会用于后续 `sourceRegex` 或正文提取。常用于模拟点击或解密。
- **sourceRegex**：资源正则，用于嗅探媒体资源（如 `.*\.(mp3|mp4).*`）。
- **正文图片链接修改 headers**：可在正文中为图片添加自定义请求头（见 JS 示例）。
- **在正文后追加信息**：可使用 `###{book.name+"正文卷"+title}` 来净化标题，或者使用 `##{{chapter.title+chapter.index}}` 来净化。

## 四、URL 参数与请求头

### 4.1 请求头配置
- 全局 `header` 为 JSON 字符串。
- 支持代理：`"proxy": "socks5://127.0.0.1:1080"` 或 `"proxy": "http://127.0.0.1:1080"`，支持用户名密码 `socks5://user:pass@host:port`。
- 可用 JS 动态生成 header（注意返回 JSON 字符串）。

### 4.2 GET/POST 请求（含 charset, webView）
在 URL 后面添加 `,{...}` 选项，格式为 JSON 对象：

```json
"https://example.com,{ \"charset\": \"gbk\", \"headers\": {...}, \"webView\": true }"
```

- `method`：GET/POST（默认 GET）
- `body`：POST 请求体
- `charset`：字符集（默认 utf-8）
- `webView`：非空时启用 WebView 加载（如 `true` 或 `1`）
- `headers`：请求头对象
- `js`：在访问 URL 前执行的 JS（如修改 `java.url`）
- `retry`：重试次数（默认 0）

### 4.3 URL 参数 `UrlOption` 完整字段
```kotlin
data class UrlOption(
    val method: String?,      // GET/POST
    val charset: String?,     // 字符集
    val webView: Any?,        // 是否启用 WebView
    val headers: Any?,        // 请求头
    val body: Any?,           // POST 数据
    val type: String?,        // 类型
    val js: String?,          // 执行 JS
    val retry: Int = 0        // 重试次数
)
```

### 5.2 `java` 对象完整方法列表

#### 网络请求
- `java.ajax(urlStr: String, callTimeout: Int? = null): String` – 同步 GET 请求，返回字符串。

### 5.8 导入 Java 类与 `JavaImporter`

使用 `JavaImporter` 可以引入 Java 包，简化调用：

```js
var javaImport = new JavaImporter();
javaImport.importPackage(
    Packages.java.lang,
    Packages.java.security
);
with (javaImport) {
    function strToMd5(str) {
        var md5 = MessageDigest.getInstance("MD5");
        var bytes = md5.digest(String(str).getBytes());
        var sb = new StringBuilder();
        bytes.forEach(b => {
            var bt = b & 0xff;
            if (bt < 16) sb.append("0");
            sb.append(Integer.toHexString(bt));
        });
        return sb.toString();
    }
}
```

## 六、动态加载与 WebView

当页面内容由 JavaScript 动态生成时，需启用 `webView`。

### 启用方式

1. **在 URL 后添加参数**：
   - `"searchUrl": "https://example.com/search?q={{key}},{\"webView\":true}"`
   - 或 `"chapterUrl": "a@href##$##,{\"webView\":true}"`

2. **使用 JS 拼接**：
   - `"bookUrl": "@js:result + ',{\"webView\":true}'"`

3. **利用 `{{}}` 拼接**：
   - `"bookUrl": "{{@@tag.a@href}},{\"webView\":true}"`

4. **在 `webJs` 中执行 JS**（见正文规则）。

### 注意事项
- 启用 WebView 会显著降低速度，仅当确实需要时使用。
- 在 `webJs` 中必须有返回值（不为空），否则会无限循环。
- 资源嗅探时，章节链接加 `,{"webView":true}`，然后在 `sourceRegex` 中填写资源正则（如 `.*\.(mp3|mp4).*`），正文填写 `<js>result</js>`。

## 九、常见问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| CloudFlare 验证 | 用 `java.startBrowser` 手动验证或使用 WebView |
| 内容“加载中” | 使用 WebView（加 `,{"webView":true}`） |
| 目录顺序倒序 | 在 `chapterList` 前加 `-` |
| 正文含广告 | 使用 `@ownText` 或正则净化（`##广告词##`） |
| 需要分页 | 配置 `nextTocUrl` / `nextContentUrl` |
| 图片防盗链 | 添加 `Referer` 请求头 |

| 搜索无结果 | 检查 `checkKeyWord` 或选择器是否正确 |
| 封面不显示 | 拼接完整 URL 或添加 Referer |

## 十、最佳实践与调试技巧

### 10.1 调试功能（善用）

阅读内置调试工具，可在书源编辑界面测试：

- **调试搜索**：输入关键词（如“系统”）查看搜索结果。
- **调试发现**：输入发现 URL（如 `月票榜:https://...`）查看发现列表。
- **调试详情页**：输入详情页 URL（如 `https://m.qidian.com/book/xxx`）。
- **调试目录页**：输入目录页 URL（前加 `++` 如 `++https://.../toc`）。
- **调试正文页**：输入正文页 URL（前加 `--` 如 `--https://.../chapter/123`）。

### 10.2 显示 JS 报错信息

在 JS 规则中使用 try-catch 捕获异常并返回错误信息：

```js
(function(result){
    try {
        // 处理 result
        return result;
    } catch(e) {
        return "" + e;  // 字符串结果返回错误信息
        // 若返回列表则 return [""+e];
    }
})(result)
```

### 10.3 代码规范与性能
- 优先使用 CSS 选择器（性能好、易读）。
- 避免滥用 WebView。
- 合理使用缓存（`cache.put`）。
- 在复杂规则中添加注释。
- 注意 Cookie 安全，不要公开含敏感信息的规则。
